ScrollableControl 的用途

子控件的概念

假如有一个对话框控件，它内部可能显示了其它控件，内部的这些控件就是这个对话框控件的子控件。子控件可以是任何 HTML 元素(nodeType=1)(如 Dom.parse("<div />")) 或 Control 对象(如 new TextBox())。在理论上，任何控件都可以有无限个子控件。比如下拉框控件中，每个菜单项就是它的子控件。控件需要对它的子控件进行增加和删除。

ScrollableControl 的功能

为了统一子控件管理的方式，比如增加一个子控件，都使用 add 函数。所有需要管理子控件的控件都必须继承 ScrollableControl 类。然后外部统一使用 ScrollableControl#add 添加子控件，而内部可以通过重写 ScrollableControl#doAdd 来实现个性化的子控件添加需求。

ScrollableControl 是管理子控件的抽象模型，当需要对子控件进行增删改查操作时。

用户可以继承 ScrollableControl 并实现个性化的子控件管理方式。当用户执行 .add 时，程序会依次调用：

  • a.add("<div />")
  • a.append("<div />")
  • a.append(dom = Dom.parse("<div />"))
  • a.insertBefore(dom, null)
  • dom = a.initChild(dom) - 子类重写此函数实现预处理子控件。
  • a.onAdding(dom, null) - 实际的插入操作，会调用 dom.attach(a.node, null)
  • a.onAdded(dom, null) - 触发 added 事件
  • return dom

当用户执行 .remove 时，程序会依次调用：

  • a.remove(dom)
  • a.removeChild(dom)
  • a.onRemoving(dom) - 实际的插入操作，会调用 dom.detach(a.node)
  • a.onemoved(dom) - 触发 removed 事件
  • return dom
